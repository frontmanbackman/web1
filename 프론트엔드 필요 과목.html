html

html은 웹페이지의 큰 뼈대를 제공한다.

html은 hypertext markup language의 약어로 웹 문서를 만들기 위하여 사용하는 기본적인 웹 언어의 한 종류이고 하이퍼 텍스트를 작성하기 위해 개발됐다.

html에서는 문서가 별도의 코드를 인식하여 완벽한 하이퍼텍스트를 만들 뿐만 아니라 단어 또는 단문을 인터넷의 다른 장소나 파일로 연결시킬 수 있다.


css

css는 색깔이나 글씨체와 같은 디자인 요소를 관리한다.

cascading style sheets의 약어로 웹 문서의 전반적인 스타일을 미리 저장해 둔 스타일시트이고 문서 전체의 일관성유지가 가능하다.

기존의 html은 웹 문서를 다양하게 설계하고 수시로 변경하는데 많은 제약이 따르는데 이를 보완하기 위해 만들어진 것이 스타일 시트이고 스타일 시트의 표준안이 바로 CSS이다.

javascript

javascript는 크로스 플랫폼, 객체지향 스크립트 언어로 웹페이지의 동작을 담당한다.

javascript는 컴파일 과정이 없기 때문에 다른 언어와 비교했을 때 빠른 시간 안에 스크립트 코드를 작성할 수 있게 도와준다.

javascript는 웹에 특화된 기술이기 때문에 운영체제나 플랫폼에 상관없이 잘 작동된다. 단점은 성능이나 보안 측면인 일단 내부에서 제공되는 기능이 제한적이고, 관련된 개발도구도 적은 편이다.

react

react는 사용자 인터페이스를 만들기 위한 자바스크립트 라이브러리다.

페이스북이 만든 기술로 커스텀 태그, 가상 dom, 단방향 데이터 바인딩 기능을 제공하고 react기술 중 일부는 모바일 쪽에서 활용되고 있다.

angular

angular는 웹 애플리케이션 프레임워크로 개발과 테스트 환경을 단순화시킨 기술이다.

angular는 제이쿼리나 자바스크립트 UI 컴포넌트를 쉽게 재사용할 수 있고 양방향 데이터 바인딩을 통해 불필요한 코드를 제거해 향후 유지보수를 쉽게 할 수 있는 장점도 있다.

jquery

jquery란 화면의 동적 기능을 자바스크립트보다 더 쉽고 편리하게 개발할 수 있게 해주는 자바스크립트 기반라이브러리다.

jquery는 여러가지 효과나 이벤트를 간단한 함수 호출만으로 빠르게 개발할 수 있고 메서드 체인 방식을 수행해 여러개의 기능이 한 줄로 나열되어 코드가 불필요하게 반복되는 것을 피하게 해준다.

css선택자를 사용해 각 html 태그에 접근해서 작업하므로 명료하면서도 읽기 쉬운 형태로 표현할 수 있다.

jsp

javaserver pages의 약어로 프로세스 방식이 아닌 스레드 방식으로 실행한다.

jsp는 html내에 자바 코드를 삽입하여 웹 서버에서 동적으로 웹 페이지를 생성하여 웹 브라우저에 돌려주는 서버 사이드 스크립트 언어이다.

jsp는 클라이언트의 요구를 처리하는 기능을 최초 한 번만 메모리에 로드 후 클라이언트가 동일한 기능을 요구하면 기존에 사용한 기능을 재사용한다.

typescript

웹 클라이언트와 서버 프로그램 개발에 모두 사용되는 오픈 소스 프로그래밍 언어이다.

javascript의 상위 언어로 angular, react, node.js등의 자바스크립트 기반 프로그램은 타입스크립트 프로그램으로도 동작한다.

구글의 공식 개발 언어로 채택되어 구글의 클라이언트 개발 환경에서 무제한 사용할 수 있다.

vue.js

vue.js는 자바 스크립트 웹 프레임워크이다.

데이터가 표현되는 view 영역과 model 영역 및 처리 로직을 어느 정도 분리할 수 있다.

양방향 바인딩 등을 적용함으로 코드양을 획기적으로 줄일 수 있어 빠른 생산성을 가진다.

컴포넌트 별로 분리할 수 있으므로 프로젝트에서 코드 재사용성이 좋다.

nuxt.js

vue.js를 더욱 손쉽게 개발하기 위한 프레임워크이다.

서버사이드 렌더링 방식으로 초기 렌더링 속도를 줄이고 동적으로 바뀌는 부분만 사용자의 접근에 따라 유동적으로 바꿔준다.

tomcat

apache software foundation에서 서버인 자바를 움직이게 하기 위해 개발한 ap서버이다.

웹프로그램은 순수한 프로그램 언어로만 이루어진 것이 아니라 클라이언트 언어인 html이 혼용되었는데 톰캣은 자바 파일만 해석할 수 있다.

웹서버에서 servlet, jsp, 빈즈와 같은 자바 코드를 이해하기 위해서는 톰캣 같은 자바를 이해하는 엔진이 있어야 한다.

xml

인터넷 웹페이지를 만드는 html을 획기적으로 개선하여 만든 언어이다.

홈페이지 구축기능 검색기능 등이 향상되었고 웹 페이지의 추가와 작성이 편리해졌다.

html은 웹 페이지에서 데이터베이스처럼 구조화된 데이터를 지원할 수 없지만 xml은 사용자가 구조화된 데이터베이스를 뜻대로 조작할 수 있다. 구조적으로 xml 문서들은 sgml 문서 형식을 따르고 있다.

java class

자바에서는 어떤 프로그래밍이든 클래스안에 속해 있다.

다른 클래스에서 다른 클래스를 선언하여 사용할 수 있다.

사물 하나하나를 기능별로 묶어서 사용하는것을 의미하고 각 클래스 안에서 역할에 따라 각 클래스의 기능을 서술해 나간다.


wordpress

오픈소스를 기반으로 한 설치형블로그 또는 컨텐츠 관리 시스템이다.

php기반이며 전세계 웹사이트의 41.6프로가 워드프레스로 제작되었다.

전세계 거의 모든 국가에 사용자가 존재하기 때문에 외국어가 가능하다면 현지근무가 용이하지만 프로그래밍언어에 대한 지식이 필요하고 보안적인 취약점이 드러나있다.
